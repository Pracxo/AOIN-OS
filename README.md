# AOIN OS

AOIN OS is a clean-slate operating workspace.

All implementation instructions, planning notes, build notes, and future project artifacts should live inside this directory so the project remains self-contained and easy to reset, archive, or publish.

## Local Structure

- `exchange/`: realtime instruction queue for Codex work.
- `instructions/`: standing implementation rules, architecture notes, and build instructions.
- `build/`: future generated build artifacts or release notes.
- `archive/`: retired instructions, snapshots, or historical notes.
