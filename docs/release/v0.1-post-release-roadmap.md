# AION v0.1 Post-Release Roadmap

1. AION-080 v0.1 local freeze and tag preparation
2. AION-081 post-tag integrity audit and CI hotfix stabilization
3. AION-082 module ecosystem strategy
4. AION-083 activation request design, still disabled
5. AION-084 generic knowledge module metadata package
6. AION-085 non-executing deterministic module mock runtime
7. AION-086 model provider adapter hardening
8. AION-087 operator UI decision
9. production auth design
10. deployment profile design

Module strategy starts after v0.1 Brain freeze.

Future vertical modules must plug into Brain through extension, binding,
conformance, action proposal, policy, audit, provenance, operator review,
sandbox, release, and freeze gates.

No domain module should modify Brain core.

Activation remains disabled until future design gates pass. Code loading
remains disabled. Package installation, dynamic route registration, full
autonomy, external calls, controlled handoff, and production deployment logic
remain disabled.

The first governed module class is Generic Knowledge Intelligence, selected as
metadata-only because it can demonstrate retrieval, grounding, citation,
summary, explanation, and answer contracts without runtime activation risk.
