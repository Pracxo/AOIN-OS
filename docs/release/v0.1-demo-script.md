# AION v0.1 Demo Script

## Operator Narration

AION means Adaptive Intelligence Orchestration Nexus. AION OS v0.1 is the local
Brain release candidate: a deterministic governance, memory, policy, planning,
operator, and release-readiness kernel for future modules.

The Brain owns contracts, policy gates, setup checks, golden path scenarios,
release smoke, release candidate evidence, operator status, extension intake
metadata, module binding metadata, conformance readiness, and SDK/CLI access.

v0.1 intentionally keeps high-impact behavior disabled. It does not enable
production authorization, full autonomy, extension code loading, capability
activation, external notification connectors, hard delete, or controlled action
handoffs.

The golden path proves local deterministic scenario wiring with synthetic data.
It is not a workload benchmark and it does not claim production operation.

Extension intake proves a future module can be described as metadata without
loading code. Module binding and conformance prove that staged module metadata
can be reviewed before activation exists.

The RC gate proves that local checks, release smoke, setup doctor, golden path,
freeze, and package dry-run evidence can be aggregated into one release
candidate report.

The freeze gate will prove that the final v0.1 state is ready for a local tag
only after all no-go conditions are clear.

After v0.1, domain modules can be designed as separate governed modules that
plug into the Brain through extension, binding, conformance, action proposal,
and policy gates. No domain module should modify Brain core.
