- Known limitations accepted.

## Operator Acceptance Criteria

An operator can start the stack, verify health, run setup doctor, run golden
path, run RC gate, inspect operator overview, validate a metadata-only
extension manifest, and shut the stack down without external services.

## Known Intentional Limitations

v0.1 does not enable production authorization, full autonomy, extension code
loading, capability activation, external notification connectors, hard delete,
controlled action handoffs, or production deployment.

## Post-Release Next Actions

After the local v0.1 release baseline, proceed to AION-082 module ecosystem
strategy on `phase/post-v0.1-module-strategy`. Do not move, delete, or
recreate `aion-v0.1.0` during post-v0.1 planning.

AION-082 is docs, roadmap, activation design, threat model, module selection,
checklist, and metadata-only manifest work. It does not add database tables,
API routes, SDK resources, CLI commands, runtime activation, code loading,
package installation, external calls, full autonomy, controlled handoff, or
production deployment logic.

The first selected module class is Generic Knowledge Intelligence. Activation
remains disabled, code loading remains disabled, and modules must not modify
Brain core.
