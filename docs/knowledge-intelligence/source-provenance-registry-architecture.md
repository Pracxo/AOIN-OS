# Source Provenance Registry Architecture

AION-206 authorizes AION-207 to design an append-only source and provenance registry. The registry may hold metadata envelopes, immutable snapshot digests, lineage edges, citation references, policy decisions, and operator review items. It must not persist source bodies, verify claims, promote knowledge, mutate beliefs, execute network fetches, or register runtime API, CLI, SDK, workflow, migration, connector, or model-provider surfaces.

Authorization transaction: `AION-206-KI-0002`. Scope: `append-only-immutable-source-snapshot-provenance-lineage-citation-registry-core`. Formal closeout: `AION-208`.
