# 0073: Post-v0.1 Module Ecosystem Strategy

## Status

Accepted.

## Context

AION Brain v0.1.0 is the frozen local-first baseline. Future modules need a
safe strategy before AION adds runtime activation, code loading, package
installation, dynamic routes, or external calls.

## Decision

Start the post-v0.1 module ecosystem strategy.

Brain core remains frozen after v0.1 except release-blocking fixes and
architecture-approved platform changes.

The first selected module is Generic Knowledge Intelligence.

AION-082 is design and metadata only.

Activation, code loading, dynamic routes, and package installation remain
disabled.

## Reason

AION needs safe module strategy before building domain modules. The strategy
must preserve the existing Brain gates and prove that modules enter through
metadata, contracts, bindings, conformance, readiness, policy, audit, and
operator review before runtime work begins.

## Consequence

Future modules must enter through Extension Registry, Module Slots, Capability
Bindings, Conformance, Readiness, Policy, Audit, and Operator Review.

The first module path can be reviewed without executing code or expanding the
runtime boundary.

## Constraints

- No runtime activation.
- No code loading.
- No dynamic route registration.
- No package installation.
- No external calls.
- No domain-specific module logic in Brain core.
