# ADR 0072: v0.1 Release Freeze Baseline

## Status

Accepted.

## Decision

Freeze AION Brain v0.1.0 as the local release baseline.

AION-080 adds release closure only, not a new subsystem. Final scripts aggregate
existing gates. v0.1 remains local-first and dry-run safe. Disabled boundaries
remain disabled.

## Reason

AION needs a stable Brain baseline before domain module work begins. The release
closure gives operators a repeatable local path for verification, evidence,
freeze review, and tag preparation without changing the Brain architecture.

## Consequence

Post-v0.1 work starts from a tagged Brain baseline. New module ecosystem work
must use extension, binding, conformance, action proposal, policy, sandbox, and
operator gates instead of modifying Brain core.

## Constraints

- No deployment.
- No external calls.
- No production readiness claim.
- No domain module logic.
- No extension code loading.
- No capability activation.
- No dynamic route registration.
- No hard delete.
