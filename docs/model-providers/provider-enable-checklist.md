# Provider Enable Checklist

This checklist is intentionally non-executable in AION-086.

Before a future task can enable any real provider path, it must prove:

- provider profile is reviewed
- prompt egress previews are redacted
- model input manifests are validated
- provider readiness assessment is not blocked
- output governance is required
- tool intents are blocked before execution
- grounding is required where claims are produced
- audit and provenance records are present
- operator review has happened
- credentials have an approved future design
- external model calls are explicitly enabled by a later milestone

AION-086 does not satisfy the final two items by design.
