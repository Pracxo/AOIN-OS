# Codex Workflow

- Use one task per Codex instruction.
- Run narrow tests for the changed area before broader suites.
- Run the quality gates before handoff.
- Update ADRs for architecture-changing work.
- Keep Brain core domain-neutral.
- Preserve adapter boundaries.
- Paste a summary after each task with files changed, tests run, and failures.
- Do not do broad refactors without focused tests.
- Do not skip failures silently.
